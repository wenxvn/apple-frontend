# 架构决策记录

这里只记录未来工作会依赖、且不容易反转的决定。日常步骤和小修改继续写入 `docs/PROJECT_LOG.md`。

## ADR-001 — 使用 Vite + React 作为应用基础

- **日期：** 2026-07-16
- **状态：** 已采纳
- **背景：** 项目用于学习 React、Three.js 和 GSAP，需要快速的前端开发与构建环境。
- **决定：** 使用 Vite + React 19。
- **影响：** 本地开发使用 `npm run dev`，构建使用 `npm run build`，Vercel 项目按 Vite 配置。

## ADR-002 — 使用 Tailwind CSS 4 的 Vite 插件

- **日期：** 2026-07-16
- **状态：** 已采纳
- **背景：** 项目需要学习并验证 Tailwind CSS。
- **决定：** 使用 `@tailwindcss/vite` 和 `@import "tailwindcss"`，不增加 PostCSS 配置。
- **影响：** Tailwind 配置由 `vite.config.js` 和 `src/styles.css` 承担。

## ADR-003 — 使用分层项目上下文，避免全量扫描

- **日期：** 2026-07-16
- **状态：** 已采纳
- **背景：** 长期项目的完整历史日志会逐步变大，不适合作为每次新会话的默认输入。
- **决定：** `docs/STATE.md` 保存精简的当前状态；`docs/PROJECT_LOG.md` 保存完整的只追加历史；`docs/ARCHITECTURE.md` 保存稳定的结构与边界。
- **影响：** 新会话默认读取架构和当前状态，仅在需要历史细节时定向查询项目记录。
